# Setting up the Memory Graph in Kiro IDE

If your Kiro IDE history is showing sessions as UUIDs (e.g., `Session fad5a292`) or `New Session (Continued)` instead of the actual human-readable titles generated by Antigravity, you need to properly configure how Kiro IDE parses the Antigravity `brain` directory.

## 1. Directory Structure

Antigravity stores all session data and conversation memory in the following location:
`~/.gemini/antigravity/brain/<conversation-id>/`

Inside each conversation ID directory, you will typically find:
- `.system_generated/logs/overview.txt` (The full raw conversation transcript)
- `task.md` (If the agent was in planning mode)
- `implementation_plan.md`
- `walkthrough.md`
- `scratch/` (Temporary scripts)

## 2. Fixing the "Missing Titles" Issue

When Antigravity processes a conversation, it doesn't always write a `metadata.json` at the root of the conversation ID. To get human-readable titles, your Kiro IDE extraction logic must do the following:

**Option A: Parse the Conversation Overview**
Read the `overview.txt` file located in `.system_generated/logs/overview.txt`. The first few user messages or agent summaries usually contain the objective. Kiro IDE should run a lightweight heuristic (or an LLM call) over the first 50 lines of `overview.txt` to extract a "Session Title".

**Option B: Parse Artifacts**
If the session used "Planning Mode", there will be an `implementation_plan.md` or `task.md`. The `# Title` or first heading in these files is highly accurate for naming the session.

1. Check if `implementation_plan.md` exists.
2. Read the first line starting with `# `.
3. Use this as the session name in Kiro IDE.

## 3. Important Configuration Settings

To ensure Kiro IDE accurately syncs with Antigravity:

1. **Path Configuration**: Ensure Kiro IDE is hardcoded or configured to read the absolute path: `/Users/<username>/.gemini/antigravity/brain/`
2. **File Watchers**: Set up a file watcher in Kiro IDE for `.system_generated/logs/overview.txt`. This file updates constantly as the Antigravity agent runs.
3. **Knowledge Items (KIs)**: Antigravity also stores distilled knowledge in `~/.gemini/antigravity/knowledge/`. Ensure Kiro IDE indexes this folder, as it contains curated `metadata.json` files and reference artifacts across sessions.

## 4. Troubleshooting

* **Chat doesn't load when clicked:** 
  If Kiro IDE opens a "New Chat" instead of loading the selected history, it means Kiro IDE failed to parse the `overview.txt` for that session, or it expected a specific JSON log format that Antigravity doesn't use. **Fix**: Update Kiro IDE's parser to read plain text line-by-line from `overview.txt`, treating each line as a user or model action.

* **History is entirely empty:**
  Verify permissions. Kiro IDE must have read access to the hidden `~/.gemini/` directory. (e.g., Mac Privacy & Security -> Full Disk Access).

* **Missing Context in New Sessions:**
  If a new session in Kiro IDE isn't picking up past context, remember that Antigravity relies on KIs (`knowledge/` folder) for cross-session context. Make sure you explicitly ask Antigravity to "save this to a KI" if you want it permanently remembered, or ensure Kiro IDE injects relevant KIs as context.
